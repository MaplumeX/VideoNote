## Findings

### 1. Tailwind v4 Support — Official and Full

shadcn/ui (v4.7.0, latest) **fully supports Tailwind v4**. The CLI auto-detects it:

```
$ npx shadcn@latest info
  tailwindVersion   v4
  tailwindConfig    -
  tailwindCss       src/index.css
```

No `tailwind.config.js` is needed with v4. Configuration is done via CSS `@theme` blocks.

**Init command for existing Vite + TW4 project:**

```bash
npx shadcn@latest init -y -b radix -p nova
```

Key flags:
- `-b radix` — use Radix UI as the component base (recommended for React SPA)
- `-b base` — alternative: use Base UI (MUI's headless library), newer, less community precedent
- `-p nova` — preset name (available: nova, vega, maia, lyra, mira, luma, sera)
- `--css-variables` — use CSS variables for theming (default: true)
- `--force` — overwrite existing `components.json` if needed

### 3. Dark Mode with Tailwind v4

shadcn uses **CSS class-based dark mode** in Tailwind v4, defined via a custom variant:

```css
@custom-variant dark (&:is(.dark *));
```

This means: to activate dark mode, add the `dark` class to a parent element (typically `<html>` or `<body>`). The `:root` block defines light theme variables, `.dark` block defines dark overrides.

**Toggle dark mode in React:**

```tsx
// Simple toggle
document.documentElement.classList.toggle('dark');
```

For systematic dark mode, use a provider or the `next-themes` pattern (ported for Vite):

```tsx
// Example: useMediaQuery + localStorage approach
function useTheme() {
  const [dark, setDark] = React.useState(() => {
    if (typeof window !== 'undefined') {
      return localStorage.getItem('theme') === 'dark' ||
        (!localStorage.getItem('theme') && window.matchMedia('(prefers-color-scheme: dark)').matches);
    }
    return false;
  });

  React.useEffect(() => {
    document.documentElement.classList.toggle('dark', dark);
    localStorage.setItem('theme', dark ? 'dark' : 'light');
  }, [dark]);

  return [dark, setDark] as const;
}
```

### 6. Radix vs Base UI (Component Library Choice)

| Aspect | Radix (`-b radix`) | Base UI (`-b base`) |
|---|---|---|
| Package | `radix-ui` (unified, v1.4+) | `@base-ui/react` |
| Maturity | Long-established, large community | Newer (MUI headless) |
| Button API | `Slot.Root` for asChild pattern | `Button.Props` |
| Import style | `import { Select } from "radix-ui"` | `import { Select } from "@base-ui/react/select"` |
| Component parity | Same registry items available | Same registry items available |
| RSC directive | Includes `"use client"` | No `"use client"` |

**Recommendation**: Use Radix (`-b radix`) for this project. The project is a Vite SPA (no RSC), and Radix has broader community support. The `"use client"` directives are harmless.

### 7. Step-by-Step Setup Procedure

1. Run `npx shadcn@latest init -y -b radix -p nova` from `frontend/`
   - This modifies `src/index.css` (replaces current custom theme)
   - Creates `components.json`
   - Installs `radix-ui`, `tw-animate-css`, `shadcn`, `@fontsource-variable/geist`
   - Creates `src/lib/utils.ts` (already exists, may need `--force` to overwrite)

2. Add components: `npx shadcn@latest add button input select card badge separator dropdown-menu -y`
   - Creates files under `src/components/ui/`

3. (Optional) Customize `:root` CSS variables in `index.css` to restore the project's blue primary color:
   - Change `--primary: oklch(0.205 0 0)` (neutral black) to something like `--primary: oklch(0.55 0.2 250)` (blue)
   - Adjust `--accent` and other colors as needed

4. (Optional) Remove `@import "@fontsource-variable/geist"` and `--font-sans: 'Geist Variable'` if the project doesn't want the Geist font

5. Existing components (`AppLayout.tsx`, `NoteView.tsx`, etc.) need manual migration from raw Tailwind classes to shadcn components

## Caveats / Not Found

1. **CSS variable naming shift**: The project's current `index.css` uses `--color-primary: #2563eb` (Tailwind v4 `@theme` inline approach). shadcn uses a two-layer system: `:root` defines `--primary` (raw CSS var), then `@theme inline` maps `--color-primary: var(--primary)`. This means any custom colors should be set in the `:root` / `.dark` blocks, NOT in the `@theme inline` block.
2. **`shadcn/tailwind.css` import**: This comes from the `shadcn` npm package and provides custom variants (`data-open`, `data-closed`, `data-checked`, etc.) and animation keyframes. It is required for component styling to work.
3. **`"use client"` directive**: Some Radix-based components include `"use client"` at the top. This is a Next.js RSC directive and is completely ignored in Vite. Do not remove it (shadcn CLI would re-add it on updates).
4. **`tw-animate-css` import**: Required for animation classes (`animate-in`, `animate-out`, `fade-in-0`, `zoom-in-95`, `slide-in-from-top-2`, etc.) used by Select, DropdownMenu, Dialog, etc.
5. **No existing `components.json`**: Project has not been initialized with shadcn before. `shadcn init` will create it.
6. **Font choice**: `@fontsource-variable/geist` is added by default. If the project wants to keep its current `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` stack, the `@import` and `--font-sans` should be removed from `index.css`.
